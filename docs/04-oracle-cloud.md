# Configuración Oracle Cloud (Autonomous Database)

Base de datos donde el endpoint `POST /guias/procesar-cola` persiste los mensajes consumidos de la cola 1, en la tabla `GUIAS_PROCESADAS` (Semana 8). Esta guía parte desde cero porque la base aún no está aprovisionada.

## 1. Crear la Autonomous Database (Always Free)

1. Regístrate/ingresa en [cloud.oracle.com](https://cloud.oracle.com).
2. Menú → **Oracle Database → Autonomous Database → Create Autonomous Database**.
3. Configuración:
   - Display name / Database name: `guiasdb`
   - Workload type: **Transaction Processing**
   - Deployment: **Serverless**, marca **Always Free**
   - Database version: la más reciente disponible
   - Password del usuario `ADMIN`: defínela y guárdala
   - Network access: **Secure access from everywhere** (requiere wallet mTLS)
4. **Create**. Espera a que el estado sea **Available**.

## 2. Descargar el wallet

1. En la página de la base → **Database connection**.
2. **Download wallet** (tipo *Instance wallet*), define una contraseña para el wallet y descarga el `.zip` (`Wallet_guiasdb.zip`).
3. En la misma pantalla verás los **TNS names** disponibles: `guiasdb_high`, `guiasdb_medium`, `guiasdb_low`, etc. El alias (por ejemplo `guiasdb_high`) es el valor de `ORACLE_DB_URL`.

El backend se conecta con:

```
jdbc:oracle:thin:@guiasdb_high?TNS_ADMIN=/app/wallet
```

donde `/app/wallet` es el contenido del zip descomprimido, montado en el contenedor (ver `docker-compose.yml`).

## 3. Crear el usuario de la aplicación

Abre **Database Actions → SQL** (con el usuario ADMIN) y ejecuta:

```sql
CREATE USER GUIAS_APP IDENTIFIED BY "TuPasswordSegura#2026";
GRANT CONNECT, RESOURCE TO GUIAS_APP;
GRANT UNLIMITED TABLESPACE TO GUIAS_APP;
```

`GUIAS_APP` / su password son los secretos `ORACLE_DB_USER` / `ORACLE_DB_PASSWORD`.

## 4. Tablas

Con `spring.jpa.hibernate.ddl-auto=update` (perfil `prod`) Hibernate crea las tablas automáticamente al iniciar. Si prefieres crearlas manualmente:

```sql
-- Tabla del CRUD principal (sumativas anteriores)
CREATE TABLE GUIAS_DESPACHO (
    ID              NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY PRIMARY KEY,
    NUMERO_GUIA     VARCHAR2(50) NOT NULL,
    TRANSPORTISTA   VARCHAR2(100) NOT NULL,
    FECHA           DATE NOT NULL,
    NOMBRE_ARCHIVO  VARCHAR2(255),
    S3_KEY          VARCHAR2(500),
    ESTADO          VARCHAR2(30),
    FECHA_CREACION  TIMESTAMP
);

-- Tabla NUEVA de la semana 8: mensajes consumidos desde la cola 1
CREATE TABLE GUIAS_PROCESADAS (
    ID                   NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY PRIMARY KEY,
    GUIA_ID              NUMBER,
    NUMERO_GUIA          VARCHAR2(50),
    TRANSPORTISTA        VARCHAR2(100),
    FECHA                DATE,
    S3_KEY               VARCHAR2(500),
    ESTADO               VARCHAR2(30),
    EVENTO               VARCHAR2(30),
    FECHA_PROCESAMIENTO  TIMESTAMP
);
```

## 5. Uso del wallet en el despliegue

Opción usada por el pipeline (ver `.github/workflows/deploy.yml`):

1. Convierte el zip del wallet a base64 y guárdalo como secreto `ORACLE_WALLET_B64`:

```powershell
# Windows (PowerShell)
[Convert]::ToBase64String([IO.File]::ReadAllBytes("Wallet_guiasdb.zip")) | Set-Clipboard
```

2. En el deploy, el workflow lo decodifica y descomprime en `~/app/wallet` de la EC2, carpeta que docker-compose monta en `/app/wallet` del contenedor.

Alternativa manual: copiar el zip a la EC2 con `scp` y descomprimirlo en `~/app/wallet` una sola vez.

> Nota: el driver `ojdbc11` junto a `oraclepki` (ya incluidos en el `pom.xml`) leen el wallet directamente con el parámetro `TNS_ADMIN` de la URL JDBC; no se requiere configuración adicional de Java.

## 6. Verificación

Con la app corriendo en perfil `prod`:

```
POST /guias/procesar-cola   (token con rol GUIA_OPERADOR)
```

Luego en Database Actions:

```sql
SELECT * FROM GUIAS_APP.GUIAS_PROCESADAS ORDER BY FECHA_PROCESAMIENTO DESC;
```

Deben aparecer los mensajes que estaban en la cola `guias.cola`.
